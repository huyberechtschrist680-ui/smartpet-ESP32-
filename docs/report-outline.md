# 报告提纲

## 1. 项目背景

- 选题：ESP32-S3 智能桌宠。
- 目标：通过状态机、传感器、执行器和无线控制实现可交互桌面宠物。
- 平台：PlatformIO + Arduino + ESP32-S3。

## 2. 硬件设计

- ESP32-S3 主控和课程板配置。
- OLED 显示、双舵机、按钮、旋转编码器、光敏传感器的连接。
- GPIO47 模式切换按钮。
- 供电和共地设计，特别是舵机与 WiFi 工作时的电流峰值。

## 3. 软件架构

- 状态机核心：`PetState`、`PetInput`、`PetEventQueue`。
- 主循环调度：输入读取、状态更新、输出应用。
- 硬件适配层：`hardware_io_real.cpp`。
- 通信层：BLE 控制和网站 `/sync` 控制。
- 网络任务：WiFi、HTTPS、天气和网站同步放入独立 FreeRTOS 任务。

## 4. 功能实现

- 摸头、喂食、心情、睡眠和动作控制。
- 串口命令控制。
- BLE 手机网页控制台。
- 网站控制台：心跳、状态、ACK 和命令下发。
- 天气显示页。

## 5. 测试与验证

- `pio run` 编译结果。
- 串口日志。
- OLED 显示测试。
- 按钮、编码器、光敏测试。
- BLE 网页连接和命令测试。
- 网站 `/sync` 在线状态、状态更新和命令下发测试。

## 6. 问题与优化

- BLE 与 WiFi 同时使用的资源压力。
- 阻塞式 HTTP 对主循环的影响，以及网络任务隔离方案。
- WiFi/HTTPS 电流峰值和供电稳定性。
- 后续可优化：拆分 `NetworkTask.cpp`、整理天气模块命名、补充更多实物测试记录。

## 7. 总结

- 已完成的核心功能。
- 系统模块化设计的优点。
- 后续改进方向。
